Base

  1. git config --global user.name blabla - встановлює глобально для всіх проектів ім'я, яке буде відображатися в полі автора у виконаних автором комітах.
  2. git config --global user.email blabal@bla - встановлює глобально для всіх проектів електронну адресу,яка буде відображатися в інформації про створенні коміти.
  3. git init - створення нового репозиторія git в поточній директорії, або ж прописати щлях самостійно.
  4. git status - показує стан файлів в робочій області і в індексі, які файли змінені, але ще не додані в індекс(staging area), які очікують бути закоміченими,а також є підказки, які дії необхідно провести для цього
  5. git add - додає вміст робочої директорії в індекс(staging area) для подальношого створення коміта(скріншота нашого проекта).
  6. git commit - бере всі дані із індекса(staging area) зберігає знімок у внутрішню базу даних, тобто робить знімок поточного стану проекта.
  7. git log - використовується для перегляду історії комітів, починаючи з поточного і до джерел проекту. По замовчуванню, вона показує лише історію поточної гілки, але може бути налаштована на вивід історії інших.
  8. git checkout- використовується для переключення гілок і вигрузки їх вмісту в робочу директорію.
  9. git branch - відображає список гілок проекту, а також вказує на поточну.
  10. git push - встановлення зв'язку з віддаленним репозиторієм, відслідковування локальних змін відсутніх в ньому і відповідно відправки цих змін на віддалений репозиторій.
  11. git pull - дозволяє загрузити останню версію із віддаленого репозиторію і обновити локальну версію.
  12. git merge - використовується для злиття однієї або декількох віток з поточною.
  13. git clone - клонує віддалений репозиторій разом із усією історією змін в локальний каталог
